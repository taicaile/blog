    To list all the shells on a Linux system,

    1
    cat /etc/shells

    The default shell in a Linux system can be found in the passwd file,

    1
2
    sudo cat /etc/passwd | grep "home/$USER"
sudo cat /etc/passwd | grep ~

    Bash startup files

    • Invoked as an interactive login shell

    All scripts use non-interactive shells.

    An interactive shell generally reads from, and writes to, a user's terminal: input and output are connected to a terminal.

    Is this shell interactive? Test by the special parameter -, it contains an i when the shell is interactive.

    1
2
3
    echo $-
himBHs
[[ $- == *i* ]] && echo 'Interactive' || echo 'Not interactive'

    In non-interactive shells, the prompt, PS1, is unset. To check if you are in an interactive shell:

    1
    [[ $- == *i* ]] && echo 'Interactive' || echo 'Not interactive'

    3. Which shell configuration files are read when you login to your system using the graphical user interface and then opening a terminal window?

      The ~/.bashrc file.

    4. Check the answers below:

      A shell opened by clicking on the background of your graphical desktop, selecting "Terminal" or such from a menu.

      It is interactive shell, but not login shell.

      A shell that you get after issuing the command ssh localhost.

      It is interactive shell and login shell.

      A shell that you get when logging in to the console in text mode.

      It is interactive shell and login shell.

      A shell obtained by the command xterm &.

      It is interactive shell, but not login shell.

      A shell opened by the mysystem.sh script.

      It is neither interactive shell nor login shell.

      A shell that you get on a remote host, for which you didn't have to give the login and/or password because you use SSH and maybe SSH keys.

      It is interactive shell, but not login shell.

    5. Can you explain why bash does not exit when you type Ctrl+C on the command line?

      The shell is in interactive mode, the error will be hanldled by the interpreter.

    6. Display directory stack content.

      1
2
3
      dirs
# or
echo $DIRSTACK

    7. If it is not yet the case, set your prompt so that it displays your location in the file system hierarchy, for instance add this line to ~/.bashrc:

      1
      export PS1="\u@\h \w> "

      Check this prompt special character for reference,

      https://www.gnu.org/software/bash/manual/html_node/Controlling-the-Prompt.html

    8. Display hashed commands for your current shell session.

      By executing the command hash,

      1
      hash

    Getting sudo to work for an alias

    1
    alias sudo='sudo '

    From man bash:

    If the last character of the alias value is a blank, then the next command word following the alias is also checked for alias expansion.
